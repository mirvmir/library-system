# Пользовательская документация веб‑сайта электронного книжного магазина

## 1. Общее описание

Веб‑приложение представляет собой электронный книжный магазин с возможностью поиска, покупки и управления книгами. Система поддерживает две роли:

* Пользователь (User) — покупает и просматривает книги
* Администратор (Admin) — управляет каталогом и заказами

---

## 2. Роль: Пользователь

### 2.1. Просмотр каталога книг

Пользователь может:

* Просматривать список книг
* Видеть:

  * название
  * цену
  * наличие
* Сортировать книги по:

  * названию
  * цене
  * доступности

### 2.2. Поиск книг

В верхней части интерфейса доступно поле поиска.

Функции:

* Поиск по названию
* Быстрая фильтрация списка

---

### 2.3. Добавление в корзину

Пользователь может:

* Добавить книгу в корзину
* Удалить книгу из корзины

Корзина отображает:

* список выбранных книг
* итоговую стоимость

---

### 2.4. Оформление заказа

Пользователь может:

* оформить заказ из корзины
* подтвердить покупку

После оформления:

* заказ сохраняется в системе

---

### 2.5. Просмотр заказов

Пользователь может:

* просматривать историю заказов
* видеть статус заказа

---

### 2.6. Работа с недоступными книгами

Если книга отсутствует:

* пользователь может оформить запрос
* система отображает ожидаемую дату доставки

---

## 3. Роль: Администратор

### 3.1. Управление каталогом книг

Администратор может:

* добавлять книги
* редактировать книги
* удалять книги

Параметры книги:

* название
* цена
* наличие

---

### 3.2. Управление поставками

Администратор может:

* работать с недоступными книгами (stale books)
* указывать дату поставки
* изменять цену

---

### 3.3. Обработка заказов

Администратор может:

* просматривать список заказов
* менять статус заказов

---

### 3.4. Работа с запросами пользователей

Администратор может:

* видеть запросы на отсутствующие книги
* обрабатывать их

---

### 3.5. Аналитика

Администратору доступна статистика:

* количество заказов
* сумма заработанных средств
* количество выполненных заказрв

---

## 4. Общие функции интерфейса

### 4.1. Навигация

Используются вкладки (Tabs):

* каталог
* корзина
* заказы
* администрирование (для администратора)

### 4.2. Сортировка

Система поддерживает сортировку:

* по названию
* по цене
* по доступности
* по дате поставки

---

### 4.3. Обновление данных

Кнопка обновления позволяет:

* перезагрузить данные каталога
* актуализировать заказы

---

### 4.4. Авторизация

Система поддерживает:

* вход пользователя
* разграничение ролей (User / Admin)

---

## 5. Основные пользовательские сценарии

### Сценарий 1: Покупка книги

1. Пользователь открывает каталог
2. Выполняет поиск
3. Добавляет книгу в корзину
4. Переходит в корзину
5. Оформляет заказ

---

### Сценарий 2: Работа с отсутствующей книгой

1. Пользователь ищет книгу
2. Видит, что книга отсутствует
3. Оставляет запрос
4. Получает информацию о поставке

---

### Сценарий 3: Работа администратора

1. Администратор входит в систему
2. Открывает раздел управления
3. Добавляет книгу
4. Обрабатывает заказы
